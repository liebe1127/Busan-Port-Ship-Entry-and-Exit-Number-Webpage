웹 프로그래밍 프로젝트
HTML, CSS, JS를 활용하여 동적 웹 페이지를 구현하는 프로젝트입니다. 자유 주제로 진행되며, 아래 명시된 제한 사항 및 요구 사항을 반드시 준수하여야 합니다.

1 기술 스택 및 파일 구성
프로젝트는 HTML, CSS, JS 세 가지 기술을 모두 포함해야 합니다.

HTML 파일 구성
index.html을 포함하여 최소 3개 이상의 HTML 파일이 필요합니다.
필수 파일 목록 및 역할은 아래와 같습니다.
index.html: 프로젝트의 메인 페이지
about.html: 프로젝트 및 개발자 소개 페이지
content.html: 메인 페이지에서 접근 가능한 주요 데이터 출력 페이지
마크업 시멘틱 태그 사용
header, nav, main, section, article, footer 등의 시멘틱 태그를 목적에 맞게 적극적으로 사용
단순 div와 span 위주의 마크업은 감점 요인
제공된 모든 HTML 파일은 공통된 디자인 및 구조를 가져야 합니다.
index.html, about.html, content.html은 모두 동일한 헤더, 푸터, 네비게이션 바를 가져야 함
제공된 모든 파일은 서로 링크되어야 하며, 손쉽게 접근할 수 있어야 함
2 데이터 처리 및 동적 화면 구성
content.html 페이지의 핵심 콘텐츠는 외부 데이터 파일을 활용하여 동적으로 생성되어야 합니다.

데이터 포맷: 제공되는 데이터는 반드시 JSON 형식이어야 합니다. JSON 이외의 방식(텍스트 파일, 자바스크립트 변수에 직접 할당 등)을 사용할 경우 감점됩니다. 제출시 JSON도 함께 제공되어야 합니다.
데이터 수집: 프로젝트 주제에 적합한 데이터를 스스로 수집하고 가공하여 JSON 파일로 준비해야 합니다. (공공데이터포털 등 활용)
API 활용: 외부 API 사용은 허용됩니다. 단, 평가 시점에서 API 키 만료나 서버 문제로 작동하지 않을 경우 기본 점수 0점 처리됩니다. 이를 방지하기 위해 로컬에 예비용 JSON 파일을 함께 제출하고, API 호출 실패 시 로컬 파일을 읽어오도록 예외 처리를 구현하는 것을 권장합니다.
3 코드 분리 원칙
모든 JS 및 CSS 코드는 별도의 외부 파일로 분리하여 작성해야 합니다.

HTML 문서 내부에 <style> 태그를 사용한 스타일 지정이나 <script> 태그 내부에 직접 자바스크립트 로직을 작성하는 것을 금지합니다.
예외적으로 외부 라이브러리(jQuery)를 로드하기 위한 <script src="..."> 태그 사용은 가능합니다.
4 반응형 웹 디자인 (CSS)
PC와 모바일 기기 등 다양한 화면 크기에서 정상적으로 보이는 반응형 웹으로 구현해야 합니다.

라이브러리 사용 제한: Bootstrap, Tailwind CSS 등 CSS 관련 프레임워크나 라이브러리 사용을 엄격히 금지합니다. 오직 순수 CSS(Vanilla CSS)만 사용해야 합니다.
레이아웃 기술: 미디어 쿼리(@media)와 함께 Flexbox, CSS Grid 등 최신 레이아웃 기술을 적극적으로 활용합니다.
디자인 및 애니메이션: CSS 전환(transition) 및 애니메이션(animation) 기능을 활용하여 사용자 경험(UX)을 향상시킬 경우 좋은 평가를 받을 수 있습니다.
5 자바스크립트 (JS) 라이브러리 규정
허용 라이브러리: jQuery 4.x 버전만 사용하세요. 단, 구버전(3.x 미만)을 사용할 경우 0점 처리되므로 버전을 반드시 확인하시기 바랍니다.
금지 라이브러리: jQuery를 제외한 React, Vue 등 다른 자바스크립트 라이브러리나 프레임워크의 사용은 일절 금지됩니다. 순수 자바스크립트(Vanilla JS) 작성 능력을 평가하기 위함입니다.
6 주제 검색 및 JSON 데이터 확보 방법
공공데이터포털 등에서 주제와 관련된 데이터셋을 CSV 형식으로 다운로드합니다.

파이썬(Python)의 csv, json 모듈 또는 pandas 라이브러리를 활용하여 CSV 데이터를 JSON 배열 형태로 변환합니다. 필요한 경우 해당 파이썬 변환 코드도 프로젝트 폴더 내에 함께 제출할 수 있습니다.
